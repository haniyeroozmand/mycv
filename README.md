# پورتفولیو — امان ورما (Next.js + Tailwind CSS)

یک پورتفولیوی تک‌صفحه‌ای، دقیقاً مطابق طرح، با دو تم (دارک/لایت شیشه‌ای)، دو زبان (انگلیسی/فارسی با RTL)، و انیمیشن‌ها و هاورهای نرم با Framer Motion.

## اجرا

```bash
npm install
npm run dev
```

بعد برو به http://localhost:3000

برای بیلد پروداکشن:

```bash
npm run build
npm start
```

## امکانات

- **لودینگ اولیه (Preloader):** یک صفحه‌ی لودینگ کدمحور و بامزه که چند خط کد را تایپ می‌کند، یک بیلد فیک اجرا می‌کند و آخرش «به سایت Haniye خوش اومدی!» را نشان می‌دهد. فقط یک‌بار در هر session نشان داده می‌شود (با sessionStorage). برای تست دوباره، تب را کامل ببند و باز کن یا storage را پاک کن. متن و زمان‌بندی‌اش در `components/Preloader.jsx` قابل تغییر است.
- **دو تم:** دارک (پیش‌فرض) و لایت سفیدِ شیشه‌ای (glassmorphism). در نسخه‌ی لایت گرادینت‌ها بسیار کمرنگ و سفیدی/شیشه‌ای پررنگ‌تر است. دکمه‌ی تغییر تم پایین سایدبار (و در موبایل، بالای صفحه). انتخاب کاربر در localStorage ذخیره می‌شود.
- **دو زبان:** انگلیسی (پیش‌فرض) و فارسی. با تغییر زبان، کل صفحه به RTL تبدیل می‌شود و فونت فارسی (Vazirmatn) فعال می‌شود.
- **چیدمان:** سایدبار ثابت سمت چپ + ستون اصلی + ستون اطلاعات سمت راست، عیناً مثل طرح.
- **ترتیب بخش‌ها:** Hero → About → Skills → **Featured Projects (بالا آمده)** → Experience. ستون راست: Personal Info، Testimonials، Education، Let's Connect.
- **انیمیشن:** لود تدریجی، reveal هنگام اسکرول، نوارهای مهارت متحرک، اسلایدر نظرات، هاورهای smooth با cubic-bezier نرم.
- **دسترس‌پذیری:** فوکوس کیبورد، احترام به prefers-reduced-motion، ریسپانسیو تا موبایل.

## ویرایش محتوا

همه‌ی متن‌ها، مهارت‌ها و پروژه‌ها در یک فایل است:

```
lib/content.js
```

- متن‌های دو زبان زیر `content.en` و `content.fa`.
- درصد مهارت‌ها در آرایه‌ی `skills`.
- پروژه‌ها در آرایه‌ی `projects`.
- عکس پروفایل: فعلاً از pravatar است؛ در `components/Sidebar.jsx` و `components/MobileBar.jsx` آدرس `img src` را به عکس خودت عوض کن (یا یک فایل در `public/` بگذار و مسیرش را بده مثل `/me.jpg`).

## لایو دمو — مهم بخوان

پرسیدی پروژه‌هایت روی **localhost** هستند و چطور لایو دمو بگذاری. سه راه داری:

### راه ۱ (توصیه‌شده) — پروژه‌ها را آنلاین کن
لوکال‌هاست فقط روی کامپیوتر خودت در دسترس است؛ بازدیدکننده‌ی پورتفولیو نمی‌تواند `localhost:3000` تو را باز کند. هر پروژه را رایگان دیپلوی کن و لینکش را بگذار:

- **Vercel** (برای Next.js / React عالی است): پروژه را به گیت‌هاب push کن، در vercel.com وصل کن، یک لینک مثل `https://devflow.vercel.app` می‌گیری.
- **Netlify** یا **GitHub Pages** هم گزینه‌اند.

بعد در `lib/content.js` فیلد `demo` هر پروژه را به آن لینک عوض کن:

```js
{
  title: "DevFlow",
  demo: "https://devflow.vercel.app",  // ← اینجا
  ...
}
```

### راه ۲ — اگر فعلاً نمی‌خواهی دیپلوی کنی: فقط اسکرین‌شات + گیت‌هاب
دکمه‌ی Live Demo را بردار یا به ریپوی گیت‌هاب وصل کن، و فقط با اسکرین‌شات پروژه را نمایش بده. کاربر کد را در گیت‌هاب می‌بیند.

### راه ۳ — تونل موقت (برای دموی زنده‌ی موقت)
اگر می‌خواهی همین الان لوکال‌هاست را موقتاً به اینترنت وصل کنی (مثلاً برای نشان‌دادن به یک کارفرما):

```bash
npx localtunnel --port 3000
# یا
ngrok http 3000
```

یک لینک عمومی موقت می‌گیری. این برای دمو همیشگی مناسب نیست (وقتی کامپیوترت خاموش شود لینک می‌میرد)، ولی برای نمایش لحظه‌ای خوب است.

## افکت اسکرینِ هاور (همان که خواستی)

کارت هر پروژه یک پنجره‌ی ثابت به ارتفاع ۱۷۰px است و عکس داخلش هنگام هاور به پایین اسکرول می‌شود تا کل صفحه‌ی پروژه را نشان دهد، و با کلیک، لینک Live Demo باز می‌شود.

برای بهترین نتیجه، یک **اسکرین‌شات تمام‌صفحه و بلند** از صفحه‌ی اصلی هر پروژه بگیر و در `public/projects/` بگذار با همین نام‌ها:
`devflow.png`، `ai-studio.png`، `nexshop.png`

(الان عکس‌های placeholder بلند گذاشته شده تا افکت را ببینی؛ با عکس واقعی خودت جایگزین کن.)

نحوه‌ی گرفتن اسکرین‌شات تمام‌صفحه در کروم:
F12 → Ctrl+Shift+P → بنویس `screenshot` → «Capture full size screenshot».
